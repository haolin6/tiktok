# 三分钟成果演示视频脚本

## 视频目标

三分钟视频展示完整主链路：

```text
后台发布竞拍 -> 双用户直播间出价 -> 管理端取消 -> 成交订单 -> 模拟支付 -> 并发一致性证据
```

本地地址：

- Web：`http://127.0.0.1:3000`
- API：`http://127.0.0.1:4000`

录制前执行：

```bash
npm run check:env
npm run db:migrate
npm run db:seed
npm run dev
```

浏览器窗口：

- 窗口 1：后台 `/admin/auctions/new` 和 `/admin/auctions`。
- 窗口 2：`/live/:roomId`，选择 `竞拍用户 A`。
- 窗口 3：同一个 `/live/:roomId`，选择 `竞拍用户 B`。

## 0:00-0:15 项目定位

画面：README 顶部或演示 PPT 封面。

口播：

```text
这是实时竞拍大师，一个抖音电商直播竞拍全栈系统。当前版本完成后台发布竞拍、用户实时出价、WebSocket 同步、成交订单、模拟支付和并发一致性验证。直播画面使用本地素材模拟，支付为模拟支付。
```

## 0:15-0:45 后台发布竞拍

页面：`/admin/auctions/new`

操作：

1. 填写商品名称、图片和介绍。
2. 设置起拍价 `99`、加价幅度 `10`、封顶价 `129`、时长 `30` 秒。
3. 点击 `创建并开始`。

画面结果：

- 页面显示竞拍 ID。
- 状态进入 `Running`。
- 出现 `进入直播间` 链接。

口播：

```text
主播在后台配置商品和竞拍规则。创建并开始后，竞拍进入 Running 状态，用户可以进入同一个直播间参与出价。
```

## 0:45-1:25 双端实时出价

页面：两个 `/live/:roomId` 窗口并排。

操作：

1. 左侧选择 `竞拍用户 A`。
2. 右侧选择 `竞拍用户 B`。
3. 用户 A 出价。
4. 用户 B 出价。

画面结果：

- 两个窗口连接状态为 `connected`。
- 当前价从 `¥99.00` 到 `¥109.00`，再到 `¥119.00`。
- 领先者、最近出价和排行榜在两个窗口同步变化。

口播：

```text
直播间通过 Socket.IO 做房间级广播。一个用户出价后，另一个窗口无需刷新即可看到当前价、领先者、最近出价和排行榜更新。
```

## 1:25-1:50 管理端取消

页面：`/admin/auctions`

操作：

1. 打开一个 `Scheduled` 或 `Running` 竞拍。
2. 点击该行 `取消`。
3. 输入取消原因 `demo cancel`。

画面结果：

- 竞拍状态变为 `已取消`。
- 直播间显示取消后的终态。

口播：

```text
管理端可以取消未结束竞拍。后端复用同一个取消 API 和状态机，终态竞拍不会再次显示取消入口。
```

## 1:50-2:25 成交订单和模拟支付

页面：`/live/:roomId`、`/pay/:orderId`、`/admin/orders`

操作：

1. 使用另一个未取消竞拍继续出价到封顶价。
2. 直播间显示成交。
3. 赢家点击 `去支付`。
4. 支付页点击 `模拟支付`。
5. 打开后台订单列表。

画面结果：

- 竞拍状态为 `Sold`。
- 支付页状态从 `pending_payment` 变为 `paid`。
- 后台订单列表显示买家、金额和支付状态。

口播：

```text
达到封顶价后系统自动成交并生成订单。模拟支付会更新 MySQL 中的订单状态，并通过 WebSocket 广播 order.paid。
```

## 2:25-2:50 并发一致性

页面：终端。

命令：

```bash
npm run demo:concurrency -- --mode=unique
```

画面保留：

```text
attempts=100
accepted=1
rejected=99
finalStatus=Sold
orderCount=1
acceptedBidRows=1
rejectedBidRows=99
```

口播：

```text
并发脚本模拟 100 个用户同时抢同一口价。最终只有一个请求成功，只有一个订单生成。Redis lock 控制并发关键区，Redis 幂等 key 处理重复请求，MySQL 唯一约束兜底订单唯一性。
```

## 2:50-3:00 收束

画面：README 的技术栈和当前边界，或 PPT 架构页。

口播：

```text
当前提交版本完成了直播竞拍主链路、实时同步、模拟支付和并发一致性验证。真实直播推流、真实支付、复杂鉴权和线上千级压测没有接入。
```
