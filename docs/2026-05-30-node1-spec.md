# 2026-05-30 节点 1 详细 Spec：项目骨架 + 数据模型 + 后端基础链路

## 1. 今日目标

节点 1 只证明后端基础链路成立：

商品创建 -> 竞拍创建 -> 竞拍查询 -> 快照查询 -> 竞拍取消。

本节点不实现 WebSocket、用户出价、成交订单、自动延时、排行榜、真实登录鉴权和漂亮后台。`orders` 与 `bids` 只建表，不写业务链路。

## 2. 修改边界

允许修改：

- `package.json` 和 TypeScript/workspace 配置。
- `apps/api/**`
- `apps/web/**`
- `packages/shared/**`
- `infra/mysql/**`
- `docs/2026-05-30-node1-spec.md`

约束：

- API route 只能接收请求并调用 service，不在 route 内写状态机和 SQL 细节。
- 状态转换只通过 `packages/shared` 中的状态机函数判断。
- 迁移脚本只做 `CREATE ... IF NOT EXISTS`，不 `DROP`、不 `TRUNCATE`、不重置已有数据库。
- Seed 只写 demo 主播、3 个 demo 用户、1 个 demo 直播间，使用 `demo_key` 幂等更新。
- 测试失败时修业务逻辑或测试隔离，不放宽状态机、不绕过校验、不删断言。

## 3. 实现内容

### 工程骨架

- 根项目使用 npm workspaces。
- `packages/shared` 输出 API 类型、状态枚举和状态机。
- `apps/api` 使用 Fastify + TypeScript + mysql2。
- `apps/web` 使用 React + TypeScript + Vite，只提供最小健康页面。

### 数据模型

核心表：

- `users`
- `products`
- `auction_rooms`
- `auctions`
- `bids`
- `orders`
- `auction_events`

节点 1 API 写入 `products`、`auctions`、`auction_events`。`bids` 与 `orders` 今天仅作为后续节点的数据承载预留。

### HTTP API

- `GET /health`
- `POST /api/products`
- `POST /api/auctions`
- `GET /api/auctions`
- `GET /api/auctions/:id`
- `GET /api/auctions/:id/snapshot`
- `POST /api/auctions/:id/cancel`

规则：

- JSON 对外使用 camelCase，MySQL 字段使用 snake_case。
- 创建商品必填 `title`，可选 `imageUrl`、`description`、`createdBy`。
- 创建竞拍必填 `roomId`、`productId`、`startPrice`、`incrementStep`、`startAt`、`endAt`。
- 创建竞拍默认 `Scheduled`，`currentPrice = startPrice`。
- 取消竞拍只允许 `Draft`、`Scheduled`、`Running` -> `Canceled`。
- 重复取消或终态取消返回 409，不静默成功。
- 创建商品、创建竞拍、取消竞拍都写入 `auction_events`。

## 4. 测试与验收

环境：

```bash
npm run check:env
npm run db:mysql:ping
npm run redis:ping
```

数据库：

```bash
npm run db:migrate
npm run db:seed
```

自动测试：

```bash
npm run test
npm run build
```

手工 API 验收：

```bash
npm run dev:api
curl http://127.0.0.1:4000/health
curl -X POST http://127.0.0.1:4000/api/products \
  -H 'content-type: application/json' \
  -d '{"title":"Demo 商品","imageUrl":"https://example.com/p.png","description":"节点1手工验收"}'
```

随后用返回的 `product.id` 创建竞拍、查询列表、查询快照、取消竞拍。MySQL 中应能看到 `products`、`auctions`、`auction_events` 的对应记录。
